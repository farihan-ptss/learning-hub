🏙️ Peta Interaktif: "Bandar Komputer"

Sukar faham istilah teknikal seperti Bus dan RAM? Klik pada "bangunan" atau "jalan raya" di bawah untuk melihat analoginya!

👈 Klik mana-mana komponen bandar di atas untuk melihat bagaimana ia bekerjasama!

🏭 Kilang Besar = CPU (Central Processing Unit)

Kilang besar inilah yang memproses segala bahan mentah (data) yang masuk ke dalam bandar komputer. Kilang ini mengandungi unit pemproses utama seperti ALU (untuk mengira), Control Unit (untuk memberi arahan), dan Register (sebagai stesen kerja kecil berkelajuan tinggi).

🏢 Gudang Penyimpanan = RAM & SSD/Harddisk

Gudang besar yang menyimpan semua barangan sementara (RAM) untuk cepat dihantar ke kilang, manakala barangan kekal disimpan selamat di dalam lori stor simpanan utama (SSD atau hard disk).

🛣️ Jalan Raya Utama = System Bus

Sistem jalan raya khas yang membolehkan semua komponen (Gudang, Kilang, Port Luaran) menghantar lori membawa maklumat dan data. Tanpa jalan raya ini, komputer anda akan lumpuh kerana maklumat tidak dapat bergerak.

🚦 Penterjemah Trafik & Persimpangan = Interface Module (Contoh: USB Controller)

Komponen luar bercakap dalam "bahasa" yang berbeza dengan kilang utama (CPU). Interface module bertindak seperti penterjemah di persimpangan jalan masuk bagi memastikan semua peranti luar seperti Mouse, Pendrive, dan Keyboard dapat difahami sepenuhnya oleh CPU.

⚡ Simulasi: Langkah-langkah Membuka Google Chrome

Bagaimana komponen bekerjasama secara harmoni melalui sistem Bus? Mari lihat prosesnya secara interaktif!

Laluan Aliran Data Langkah 1 daripada 5
🖱️

Langkah 1: Klik Ikon Google Chrome

Mouse menghantar isyarat klik "Buka Chrome!" melalui Bus terus menuju ke CPU.

Mouse ➡️ BUS ➡️ CPU
🏭

Langkah 2: CPU Terima Arahan

CPU meminta fail pemasangan Google Chrome daripada SSD (Storage) dengan menghantar arahan bacaan melalui Bus.

CPU ➡️ BUS ➡️ SSD
💾

Langkah 3: SSD Hantar Fail

SSD yang menerima arahan tadi, menyalin fail perisian Chrome dan menghantarnya menerusi Bus untuk diletakkan di dalam RAM.

SSD ➡️ BUS ➡️ RAM
🧠

Langkah 4: Pemprosesan Paparan

CPU mula memproses arahan grafik daripada RAM lalu menghantar isyarat data bergambar kepada GPU (kad grafik) menggunakan Bus.

RAM ➡️ CPU ➡️ BUS ➡️ GPU
🖥️

Langkah 5: Tuala Paparan Terhasil! 🎉

GPU menghantar isyarat gambar akhir ke Monitor anda. Halaman Google Chrome kini berjaya dipaparkan di skrin!

Semua perkakasan berjaya berhubung melalui Bus Interconnection!
