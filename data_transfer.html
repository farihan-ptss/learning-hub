S = Same Clock ⏰

Synchronous Method

Penghantaran data yang menggunakan isyarat jam (clock) yang sama antara penghantar dan penerima. Kedua-dua peranti diselaraskan secara jitu untuk mengetahui bila hendak menghantar dan menerima data.

🚶🚶 Analogi Ali & Abu Berjalan:

Mereka melangkah kaki pada rentak yang sama serentak. Ali (👣 👣) & Abu (👣 👣). Tiada siapa lebih cepat atau perlahan.

🚦 Analogi Lampu Isyarat:

Semua kereta bergerak serentak apabila lampu hijau menyala secara berjadual.

Contoh Utama:
CPU ↔ RAM CPU ↔ Cache DDR RAM
Sangat Laju & Teratur Kompleks (Gantung pada Jam)
A = Anytime Ready 📱

Asynchronous Method

Penghantaran data tanpa berkongsi clock yang sama. Data hanya dihantar apabila kedua-dua peranti sudah bersedia. Sebelum data dihantar, isyarat persediaan (request & response) ditukarkan.

💬 Analogi WhatsApp:

Anda hantar mesej pukul 10.00 pagi ("Hi"). Rakan membalas pada pukul 10.30 pagi ("Hi juga"). Tiada masa tetap, masing-masing bertindak apabila lapang.

⌨️ Analogi Keyboard:

Sistem hanya menerima isyarat apabila anda menekan kekunci huruf 'A'. Tiada jam khas yang mengatur bila anda perlu menekan.

Contoh Utama:
Keyboard ↔ CPU Mouse ↔ CPU Printer ↔ CPU
Mudah & Fleksibel Perlahan (Menunggu Isyarat)

Jadual Perbandingan Pembelajaran

Ciri Perbandingan Synchronous (⏰) Asynchronous (💬)
Penggunaan Clock Menggunakan isyarat jam (clock) yang sama. Tidak menggunakan clock yang sama.
Masa Penghantaran Pada masa/fasa clock yang ditetapkan. Bila-bila masa apabila peranti sudah sedia.
Kelajuan Sangat Pantas. Sedikit lebih perlahan (ada handshaking).
Sifat Aliran Komunikasi sangat tersusun & teratur. Lebih fleksibel, rawak (on-demand).
Formula Mudah "Ikut jam yang sama." ⏰ "Hantar bila sudah bersedia." 💬
Single Lane / Lorong Tunggal 🚗

Serial Transfer

Kaedah penghantaran data di mana hanya satu bit dihantar pada satu masa melalui satu saluran wayar tunggal. Semua bit beratur dan tiba satu demi satu.

📦 Analogi Lori Hantar Kotak:

Hanya ada sebuah lori. Lori membawa 1 kotak pada satu masa. Semua 8 kotak tiba satu persatu (📦 → 📦 → 📦).

🛣️ Analogi Lebuh Raya:

Lebuh raya hanya ada 1 lorong. Kereta wajib mengekor satu demi satu.

Contoh Utama:
USB (Universal Serial Bus) SATA (SSD/HDD) Ethernet & HDMI
Kos Murah & Jarak Jauh Beratur Satu-Satu
Multi Lane / Banyak Lorong 🚗🚗🚗

Parallel Transfer

Kaedah penghantaran data di mana beberapa bit dihantar serentak menggunakan banyak saluran (wire) pada masa yang sama. Kesemua bit tiba bersama-sama.

📦 Analogi Lori Hantar Kotak:

Anda mempunyai 8 lori serentak. Setiap lori membawa 1 kotak dan kesemua lori sampai ke destinasi secara serentak (📦📦📦📦📦📦📦📦).

🛣️ Analogi Lebuh Raya:

Lebuh raya luas dengan 8 lorong. 8 buah kereta boleh dipandu bersebelahan pada masa yang sama.

Contoh Utama:
Parallel Printer Port (LPT) IDE / PATA Cable (Lama) CPU-RAM Data Bus (64-bit)
Laju (Jarak Pendek) Gangguan Isyarat & Mahal

Jadual Perbandingan Pembelajaran

Ciri Perbandingan Serial Transfer (🔵) Parallel Transfer (🟢)
Bilangan Bit/Masa 1 bit sahaja pada satu masa. Banyak bit (contoh: 8, 16, 64) secara serentak.
Bilangan Wayar Hanya 1 (atau sepasang wayar sahaja). Banyak wayar yang disusun selari.
Kelayakkan Jarak Sangat sesuai untuk jarak jauh. Hanya jarak pendek (jika panjang, herotan berlaku).
Kos Kabel Jauh Lebih Murah & Ringkas. Lebih mahal & tebal (banyak laluan).

1. Programmed I/O

CPU mengawal secara terus dan manual kesemua proses pemindahan data. CPU akan bertanya peranti berulang-ulang kali (polling / busy waiting) sehingga data tersedia.

👨🏻‍🏫 Analogi Pensyarah:

Pensyarah bergerak sendiri dari satu meja ke satu meja pelajar untuk mengutip tugasan. Beliau terpaksa berdiri menunggu pelajar tersebut menyelesaikan tugasan sehingga siap.

👉 CPU tanya device: "Ada data?"
👉 CPU tunggu (busy wait)
👉 CPU ambil & simpan dalam memori
❌ CPU tidak boleh buat kerja lain!
Panggil Bila Sedia! 📱

2. Interrupt-Initiated I/O

CPU membuat kerja asalnya terlebih dahulu. Apabila peranti I/O bersedia, ia menghantar isyarat (interrupt) kepada CPU. CPU akan menghentikan seketika kerjanya untuk memproses data I/O tersebut.

🔔 Analogi Notifikasi Telefon:

Anda sedang fokus menulis laporan. Tiba-tiba telefon berbunyi notifikasi mesej masuk. Anda berhenti sebentar untuk membalas, dan menyambung semula tugasan anda.

👉 CPU buat kerja utama yang lain
👉 Peranti hantar isyarat interrupt
👉 CPU berhenti seketika, proses data
✅ CPU lebih efisien, tidak perlu tunggu!
Kurier Hantar Terus 🚚

3. Direct Memory Access

Peranti I/O memindahkan data bersaiz besar terus ke RAM melalui pengawal DMA tanpa mengganggu CPU. CPU hanya menyediakan tetapan awal dan bebas bekerja sepenuhnya.

💼 Analogi Pengurus (Manager):

Syarikat mempunyai Pengurus Besar (CPU). Pengurus hanya mengarahkan kurier (DMA) memindahkan barang terus ke Stor (RAM). Pengurus bebas merancang strategi lain.

👉 CPU set up controller DMA
👉 DMA pindah data terus ke RAM
👉 DMA hantar isyarat selesai ke CPU
🚀 Pemindahan data saiz besar paling pantas!

Jadual Perbandingan Pembelajaran

Ciri Perbandingan Programmed I/O Interrupt-Initiated I/O Direct Memory Access (DMA)
Siapa Kawal Data? CPU Sepenuhnya Peranti + CPU Pengawal DMA (DMA Controller)
Status Keadaan CPU Sangat Sibuk (Menunggu) Sederhana Sibuk (Hanya diganggu bila sedia) Sangat Ringan / Bebas sepenuhnya
Kelajuan Pemindahan Perlahan Sederhana Sangat Laju (Sesuai fail saiz besar)
Ringkasan Ringkas "CPU pergi ambil sendiri." "Bila siap, peranti panggil CPU." "Kurier buat semua, bos goyang kaki."
